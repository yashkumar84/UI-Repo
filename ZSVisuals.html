üíªProblem 1: Two Sum

Easy ‚≠ê Most Frequently Asked

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

Example:

Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: nums[0] + nums[1] = 2 + 7 = 9
Use a HashMap to store complements. Time: O(n), Space: O(n) 
// Java Solution
public int[] twoSum(int[] nums, int target) {
    HashMap<Integer, Integer> map = new HashMap<>();
    
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        
        if (map.containsKey(complement)) {
            return new int[] { map.get(complement), i };
        }
        
        map.put(nums[i], i);
    }
    
    return new int[] {};
}

// Time Complexity: O(n)
// Space Complexity: O(n)

üîóProblem 2: Reverse a Linked List

Easy ‚≠ê Top Interview Question

Reverse a singly linked list iteratively and recursively.

Example:

Input:  1 ‚Üí 2 ‚Üí 3 ‚Üí 4 ‚Üí 5 ‚Üí NULL
Output: 5 ‚Üí 4 ‚Üí 3 ‚Üí 2 ‚Üí 1 ‚Üí NULL
Three pointers approach: prev, current, next. Remember to break and reform links! 
// Iterative Solution
public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode current = head;
    
    while (current != null) {
        ListNode next = current.next;  // Save next
        current.next = prev;            // Reverse link
        prev = current;                 // Move prev forward
        current = next;                 // Move current forward
    }
    
    return prev;
}

// Recursive Solution
public ListNode reverseListRecursive(ListNode head) {
    if (head == null || head.next == null) {
        return head;
    }
    
    ListNode newHead = reverseListRecursive(head.next);
    head.next.next = head;
    head.next = null;
    
    return newHead;
}

// Time: O(n), Space: O(1) iterative, O(n) recursive

üìäProblem 3: Kadane's Algorithm (Max Subarray Sum)

Medium ‚≠ê Classic DP Problem

Find the contiguous subarray with the largest sum.

Example:

Input: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
Output: 6
Explanation: [4, -1, 2, 1] has the largest sum = 6
At each position, decide: include current element in existing subarray or start fresh? 
// Kadane's Algorithm
public int maxSubArray(int[] nums) {
    int maxSoFar = nums[0];
    int maxEndingHere = nums[0];
    
    for (int i = 1; i < nums.length; i++) {
        // Either extend existing subarray or start new
        maxEndingHere = Math.max(nums[i], maxEndingHere + nums[i]);
        
        // Update global maximum
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

// Time Complexity: O(n)
// Space Complexity: O(1)

// Key Insight: maxEndingHere tracks best sum ending at position i

üîç First Non-Repeating Character

Easy

Find the first character that appears only once in a string.

public char firstNonRepeating(String s) {
    HashMap<Character, Integer> freq = new HashMap<>();
    
    // Count frequencies
    for (char c : s.toCharArray()) {
        freq.put(c, freq.getOrDefault(c, 0) + 1);
    }
    
    // Find first with freq = 1
    for (char c : s.toCharArray()) {
        if (freq.get(c) == 1) return c;
    }
    
    return '\0';
}
// Time: O(n), Space: O(1) - at most 26 characters

üîÑ Detect Cycle in Linked List (Floyd's Algorithm)

Easy

Use slow and fast pointers to detect if a cycle exists.

public boolean hasCycle(ListNode head) {
    if (head == null) return false;
    
    ListNode slow = head;
    ListNode fast = head;
    
    while (fast != null && fast.next != null) {
        slow = slow.next;           // Move 1 step
        fast = fast.next.next;      // Move 2 steps
        
        if (slow == fast) {
            return true;  // Cycle detected
        }
    }
    
    return false;
}
// Time: O(n), Space: O(1)

üî¢ Longest Subarray with Sum Zero

Medium

Find length of longest subarray with sum = 0 using prefix sums.

public int maxLenSubarraySum0(int[] arr) {
    HashMap<Integer, Integer> map = new HashMap<>();
    int sum = 0, maxLen = 0;
    
    for (int i = 0; i < arr.length; i++) {
        sum += arr[i];
        
        if (sum == 0) maxLen = i + 1;
        
        if (map.containsKey(sum)) {
            maxLen = Math.max(maxLen, i - map.get(sum));
        } else {
            map.put(sum, i);
        }
    }
    
    return maxLen;
}
// Time: O(n), Space: O(n)

üéØ Merge Two Sorted Linked Lists

Easy 
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode current = dummy;
    
    while (l1 != null && l2 != null) {
        if (l1.val < l2.val) {
            current.next = l1;
            l1 = l1.next;
        } else {
            current.next = l2;
            l2 = l2.next;
        }
        current = current.next;
    }
    
    current.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
// Time: O(m+n), Space: O(1)

Problem 1: Monthly Revenue with Window Functions

Given Sales table, find monthly totals and identify months exceeding annual average.

üìä Sales Table

sale_id sale_date product_id amount
12024-01-151011500

-- Step 1: Calculate monthly totals and annual average
WITH monthly_sales AS (
    SELECT 
        DATE_TRUNC('month', sale_date) AS month,
        SUM(amount) AS monthly_revenue
    FROM Sales
    GROUP BY DATE_TRUNC('month', sale_date)
),
annual_avg AS (
    SELECT AVG(monthly_revenue) AS avg_revenue
    FROM monthly_sales
)
SELECT 
    m.month,
    m.monthly_revenue,
    a.avg_revenue,
    CASE 
        WHEN m.monthly_revenue > a.avg_revenue 
        THEN 'Above Average'
        ELSE 'Below Average'
    END AS status
FROM monthly_sales m
CROSS JOIN annual_avg a
ORDER BY m.month;
Key concepts: CTEs, DATE_TRUNC for grouping by month, window functions for averages.

Problem 2: Second Highest Salary per Department

Use window functions to get the 2nd highest salary in each department.

üë• Employees Table

emp_id name department salary
1AliceEngineering90000
2BobEngineering85000
3CharlieSales75000
4DavidSales80000
5EveEngineering95000
 
-- Using DENSE_RANK() window function
WITH ranked_salaries AS (
    SELECT 
        name,
        department,
        salary,
        DENSE_RANK() OVER (
            PARTITION BY department 
            ORDER BY salary DESC
        ) AS rank
    FROM Employees
)
SELECT 
    department,
    name,
    salary
FROM ranked_salaries
WHERE rank = 2;

-- Alternative: Using ROW_NUMBER()
SELECT * FROM (
    SELECT 
        department,
        name,
        salary,
        ROW_NUMBER() OVER (
            PARTITION BY department 
            ORDER BY salary DESC
        ) AS rn
    FROM Employees
) t
WHERE rn = 2;
RANK() vs DENSE_RANK() vs ROW_NUMBER(): RANK skips numbers after ties, DENSE_RANK doesn't skip, ROW_NUMBER gives unique ranks.

Problem 3: Remove Duplicates Keeping Latest

Delete duplicate rows, keeping only the record with the latest timestamp.

-- Method 1: Using ROW_NUMBER() and DELETE
WITH cte AS (
    SELECT 
        *,
        ROW_NUMBER() OVER (
            PARTITION BY customer_id, order_id
            ORDER BY timestamp DESC
        ) AS rn
    FROM Orders
)
DELETE FROM Orders
WHERE (customer_id, order_id, timestamp) IN (
    SELECT customer_id, order_id, timestamp
    FROM cte
    WHERE rn > 1
);

-- Method 2: Create new table without duplicates
CREATE TABLE Orders_Clean AS
SELECT * FROM (
    SELECT 
        *,
        ROW_NUMBER() OVER (
            PARTITION BY customer_id, order_id
            ORDER BY timestamp DESC
        ) AS rn
    FROM Orders
) t
WHERE rn = 1;

üîê ACID Properties

Essential for database transactions

‚öõÔ∏è Atomicity

All or nothing - transaction completes fully or not at all

Example: Bank transfer must debit and credit both accounts

‚úÖ Consistency

Database remains in valid state before and after transaction

Example: Total money in system stays constant

üîí Isolation

Concurrent transactions don't interfere with each other

Example: Two people booking same seat see consistent data

üíæ Durability

Committed changes persist even after system failure

Example: Payment recorded survives server crash

üìä Database Normalization (3NF)

-- Unnormalized Orders Table
OrderID | CustomerName | CustomerEmail | Product | Price | Quantity
----------------------------------------------------------------------
1       | John         | john@mail.com | Laptop  | 1000  | 2
1       | John         | john@mail.com | Mouse   | 20    | 1

-- 1NF: Remove repeating groups, atomic values
Orders: OrderID | CustomerID | Timestamp
OrderItems: OrderItemID | OrderID | ProductID | Quantity

-- 2NF: Remove partial dependencies
Customers: CustomerID | CustomerName | CustomerEmail
Products: ProductID | ProductName | Price

-- 3NF: Remove transitive dependencies
// All non-key attributes depend ONLY on primary key

Benefits: Reduces redundancy, prevents update anomalies
Trade-off: More JOINs needed for queries

üåê TCP vs UDP

üîó TCP (Transmission Control Protocol)

  • ‚úÖ Connection-oriented
  • ‚úÖ Reliable delivery (ACK)
  • ‚úÖ Ordered packets
  • ‚úÖ Error checking
  • ‚ùå Slower (overhead)

Use for: HTTP, Email, File Transfer

‚ö° UDP (User Datagram Protocol)

  • ‚úÖ Connectionless
  • ‚úÖ Fast (low latency)
  • ‚úÖ No overhead
  • ‚ùå No delivery guarantee
  • ‚ùå No ordering

Use for: Video streaming, Gaming, DNS

‚öñÔ∏è CAP Theorem

Choose Any Two

C

Consistency

All nodes see same data

A

Availability

System always responds

P

Partition Tolerance

Works despite network failures

Examples:

‚Ä¢ CA: Traditional RDBMS (no partition tolerance)

‚Ä¢ CP: MongoDB, HBase (sacrifice availability for consistency)

‚Ä¢ AP: Cassandra, DynamoDB (eventual consistency)

üö™ Two Guards and Two Doors

Problem: One door leads to freedom, one to death. One guard always lies, one always tells truth. You don't know which is which. You can ask ONE question to ONE guard.

The Magic Question:

"If I asked the OTHER guard which door leads to freedom, 
what would they say?"

Then take the OPPOSITE door!

// Why it works:

Scenario 1: You ask the TRUTH-TELLER
- They honestly tell you what the LIAR would say
- The liar would point to DEATH door
- So you get: "They'd say the death door"
- You take opposite ‚Üí FREEDOM ‚úì

Scenario 2: You ask the LIAR
- They lie about what TRUTH-TELLER would say
- Truth-teller would point to FREEDOM door
- Liar lies and says: "They'd say the death door"
- You take opposite ‚Üí FREEDOM ‚úì

// The double-negative cancels out!
